The Wo tool

 The WoT.exe program, which is itself a Wo application, allows to:
  • build a template for a new application.
  • visualize and modify the interface of an existing application without starting this application.
  • get various help and informations.

The command syntax for the WoT tool has the form: 

    SYS> WoT [-option [parameters]]

To create a new Wo application (myapp), type: 

 
    SYS> WoT -application myapp
The tool will create: 
    . myapp.c : main C program.
    . myapp.odb : the principal resource file.
    . myapp.htm : on-line documentation for myapp.
    . Makefile : UNIX makefile to build myapp.exe.
    . descrip.mms : VMS MMS file to build myapp.exe.
To visualize the interface contained in myapp.odb, type: 
    SYS> WoT -show myapp.odb
For example, a sum up of Wo features could be seen with: 
    SYS> WoT -show WoTest.odb

You can give X11 options after -show: 

    SYS> WoT -show WoTest.odb -xrm "*borderWidth:5"

You can dump, in the Wo console, some help topic with: 

    SYS> WoT -topic "file#topic"
    SYS> WoT -topic "Wo.htm"
The topic locator follow WWW URL syntax: [#]. If topic is omitted, first one in file is taken. LaTeX file (.tex) of a topic could be produced with: 
    SYS> WoT -latex "Wo.htm" 
        --> Wo.tex
    SYS> WoT -latex "myapp.htm"
        --> myapp.tex
The production of postscript files from latex files is site dependent; contact site manager to do it.

To show Wo version: 

    SYS> WoT -version

To have addresses for WoT bug report: 

    SYS> WoT -author
Various tracing mode could be enabled with the "-trace" option: 
    SYS> WoT -trace widget -show WoTest.odb
      dump created and destroyed widgets.
    SYS> WoT -trace resource -show WoTest.odb
      dump resource informations.
    SYS> WoT -trace event -show WoTest.odb
      dump main loop actions.
    SYS> WoT -trace start -show WoTest.odb
      dump user interface startup elapsed time.
